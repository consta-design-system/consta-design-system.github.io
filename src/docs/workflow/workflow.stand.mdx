Дизайн-система — это библиотека готовых компонентов. С ней можно работать
как с любой библиотекой: подключить и использовать компоненты,
создавать новые, использовать только макеты, только код или просто
посмотреть и сделать так же — зависит от вашей задачи.

Здесь описан вариант использования дизайн-системы, который кажется
нам оптимальным.

## 1. Дизайн-макет

1. 2. Создаёт макеты интерфейсов с помощью готовых компонентов из Consta.3. Если готовых компонентов не хватает, разрабатывает свои компоненты по принципам дизайн-системы. 4. Создаёт дизайн-спецификацию для разработчиков: описывает, как ведут себя элементы, какие у них могут быть модификации, варианты цветовых схем, анимацию — всё, что может понадобиться при реализации проекта.

После этого макеты отправляются в разработку.

> Если у вас нет дизайнера,
> можете использовать дизайн-систему как брендбук: посмотреть и сделать так же.

## 2. Разработка

1. Разработчик устанавливает NPM-пакеты с библиотеками дизайн-системы. Например, `@consta/uikit` и `@consta/charts`.
2. Подключает стандартную тему со стандартной цветовой схемой (или создаёт свою
   и всё равно её включает). См. [Быстрый старт](##LIBS.LIB.STAND/lib:uikit/stand:about-start)
3. Создаёт интерфейс по макетам и спецификации, которые передал дизайнер.
   При этом использует компоненты из библиотек Consta — их работа описана в этой
   документации. [Посмотреть список компонентов](/)

Если каких-то компонентов не хватает, их можно создать самостоятельно.
Компоненты, которые вы создали для своего дизайн-проекта, можно [предложить нам](##LIBS.LIB.STAND/lib:portal/stand:custom-addcomponent). Если компоненты подойдут, их можно будет включить в дизайн-систему.

> Дизайн-система сделана на React.js. Если вы работаете с другими технологиями,
> библиотеки использовать не получится. В этом случае
> все нужные компоненты самим.

## 3. Тестирование и следующие этапы
